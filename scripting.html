I : FAQ
a ) Qu'est-ce qu'un script?
Un script est une serie de commandes. Ici nous parlerons donc de scripts ET qui contiendront des commandes déstinées à être interpretées par le jeu W:ET.
Retour au menu


b) Quel fichier dois-je modifier pour insérer des scripts?
Le fichier 'etconfig.cfg' sauvegarde votre configuration du jeu, c'est-à dire les touches et les graphismes. Ce fichier est situé dans les repertoirs 'profiles' de chaque mod.
Exemple: etmain/profil/pseudo/etconfig.cfg
Vous pourriez donc mettre vos scripts dans ce fichier mais nous vous conseillons fortement de modifier votre configuration à travers des fichiers à part car les pertes de données sur les etconfig.cfg sont fréquentes (lorsque le jeu réinitialise votre confg après une erreur par exemple). Il est même compliqué de relire un script fait après l'initialisation du jeu car W:ET classe le contenu du fichier etconfig. A la deuxième utilisation vous ne retrouverez plus vos seta ou autres au même endroit. C'est pourquoi il faut effectuer les changements dans des fichiers .cfg à part, que vous placerez dans le repertoir etmain. Il vous suffira alors de taper dans la console du jeu '/exec nom_du_fichier.cfg' pour que ces modifications s'inscrivent dans le fichier etconfig.cfg!
Un fichier bien util est 'autoexec.cfg' situé dans le repertoir /etmain. Ce fichier est éxécuté (normalement) à chaque lancement du jeu. Néanmoins on ne peut y mettre tous ses scripts car sa taille est limitée. Lorsqu'elle est atteinte, ET plante (c'est valable pour tout les scripts trop gros).
Retour au menu


c) Comment ouvrir/créer ces fichiers?
Vous pouvez ouvrir & créer les fichier '.cfg' avec WordPad.
Attention:
Un problème fréquent arrive lors de l'utilisation de WordPad, en effet, lors de l'enregistrement, celui-ci garde un format particulier, ce qui causera des erreurs lors de l'éxécution du script avec des messages tels que 'Unknow ...'!

d)Je ne m'en sors plus dans mes scripts, c'est un vrai bordel, au secours!
Pas de panique, il existe une méthode assez simple pour avoir une config "propre", même si on utilise beaucoup de scripts. Il faut créer des fichier *.cfg pour chaque script ou famille de commandes, tout en utilisant les commentaires correctement pour mieux s'y retrouver ultérieurement.
Ensuite il suffira de lancer le(s) script(s) à travers la console ET en tappant '/exec nom_du_fichier.cfg'. Le contenu s'inscrira dans 'etconfig.cfg' et vous n'aurez plus besoin de le(s) rééxécuter, mais seulement lorsque vous le(s) modifirez. Utilisez les commentaires pour bien séparer et détailler vos scripts.
Exemple: un fichier pour les bind, un autre pour le graphisme, un autre pour le changement de class/spawn, etc.

Attention:
On ne peut pas parler d'un language de programmation pour ET! Les conditions, boucles, etc. n'éxistent pas! On peut simuler des boucles avec les binds mais c'est tout.

a) Vocabulaire
FPS - "Frames par Second". Si ET affiche par exemple 50 FPS chez vous , c'est que votre machine est en mesure d'afficher 50 images par seconde. On dit que 25 FPS suffisent pour jouer aisément , "Parce qu'une télévision rafraîchie l'écran 25 fois par secondes" "... Oui, une image de télé à seulement 25 FPS , mais la technologie utilisé pour les moniteurs de PC est complètement différente et permet d'afficher des résolutions supérieures a celle d'une télévision (640*480/25fps en théorie). Dès 45 FPS, on peut parler d'un jeu suffisamment fluide pour jouer. Plus le FPS est élevé, plus votre machine traitera aisément les données, au détriment de la qualité graphique sur certaines configurations ( quantité de RAM , puissance du processeur et de la carte graphique déterminant ). Les paliers de FPS étant 45, 76, 125 et 333.
Pour voir vos FPS, tapez '/cg_drawfps 2' dans la console ET.
Pour mettre une limite à vos FPS tapez '/cg_maxfps nombre de fps max' dans la console ET.
La notion de FPS est très complexe, elle est liée au moteur physique du jeu. C'est pourquoi on retient seulement "plus j'en ai, mieux c'est!!". C'est tellement plus simple :) Vous trouverez plus d'informations ici: http://www.nofrag.com/2004/oct/23/14529/


Script - un script est un ensemble de commandes exécutées dans un ordre bien précis (pour imager dans le cadre de ET, c'est comme si j'affectais l'allumage automatique de ma télé et la fermeture des volets sur le bouton qui me sert a allumer la lumière de mon salon, ce qui serait bien pratique et me permettrai de gagner quelques minutes le soir quand je rentre tard, vivement le script-binding-domestique...). Dans un script , nous avons la possibilité de configurer une touche pour effectuer plusieurs événements ( lancer une grenade, prévenir mes équipiers de dégager le secteur et revenir à l'arme principal , le tout sur une seul touche, d'où l'exemple du bouton de lumière de mon salon). Avec cela , beaucoup de manipulations (par exemple le changement d'arme) deviennent plus facile.


Variable - Emplacement stockant une donnée, elle peut varier au cours de l'exécution du script.
Sur ET on les ecrit sous la forme: set nom_de_la_variable "données enregistrés"
Bien entendu il faut que les données enregistrées soient compréhensible par le jeu! Ceci sera traité plus bas :)
Pour appeler une variable sur ET on écrit: vstr nom_de_la_variable


bind - signifie "lier à".


set - signifie "fixer".


str - abréviation du mot "string" qui signifie en programmation "chaine de caractères".

bind - Permet d'associer une touche à une action/commande. Il éxiste beaucoup d'actions, par éxemple, tirer : +attack, se tuer: kill, etc.
Syntaxe:
bind <touche> "<action>"

Exemple:
//On veut dire bonjour en appuyant sur la touche x
bind x "say Bonjouur!"
Cf. plus bas pour les actions.
Remarque:
Les touches sont celles des claviers qwerty! Le 'a' d'un clavier azerty, par exemple, sera 'q' dans le jeu. Cf. plus bas pour avoir les équivalents azerty/qwerty.

unbind x - libère la touche "x".
unbindall - libère toutes les touches.


Wait - Cettre "fonction" est relativement complexe, mais indispensable dans beaucoup de scripts.
Syntaxe:
wait <num>

Lors de l'exécution d'un script, l'ordinateur exécute les différentes variables à une vitesse généralement trop rapide pour qu’elles s’enchaînent normalement. Par exemple si vous faites un script pour, a l'aide d'une touche, passer de votre arme a votre pack d'ammo, prendre des munitions et revenir à son arme sans utiliser "wait" : 'bind x "weaponbank 6; +attack; -attack; weaponbank 3"'.
Le jeu exécutera l'action à la voler, ce qui fera que le temps de sortir le pack d'ammo dans le jeu, le script sera déjà fini et vous vous retrouvez avec votre pack en mains. Il faut donc marquer une pause entre chaque action pour laisser le temps au jeu de l'exécuter. Là ou ça se complique, c'est que la durée du "wait" est dépendante des performances de votre machine. Si vous utilisez par exemple "wait 120", et que votre ordinateur est capable d'afficher 60 FPS ( 60 images secondes), le system va se mettre en pause pendant 2 secondes (120/60). C'est donc le rapport entre l'argument qui suit 'wait' et votre nombre de FPS au moment précis où vous appuyez sur votre touche. Si vous êtes trop gourement avec les wait vous serez bloqué durant l'éxécution, mais des wait suivient de petites valeurs ne genent pas du tout.

set - premet de déclarer une variable contenant une/des action(s)/commande(s).
Syntaxe:
set <nom> "<commande/action>"

Ceci va donc garder en mémoire/stocker le contenu <commande/action>, assigné à <nom>.
Pour redéclarer un nom déjà utilisé, il suffit de réécrire la déclaration de la variable. Si il y a plusieurs actions pour une même variable, utilisez le point-virgule ';' pour les séparer.
Pour appeler le contenu d'une variable (dans un bind ou un seta) on utilise 'vstr'.

Exemple:
// on va déclarer la variable 'nemesis' qui va remettre mon pseudo
// La variable contenant notre pseudo est 'name'
set nemesis "set name ^7Nemesis^3`^bR3"
bind x "vstr nemesis" // En fait on a bind x "set name ^7Nemesis^3`^bR3"

// Autre exemple
// On va associer à la touche x l'alternance entre Nemesis`R3 et n3mS.s`R3 comme pseudo
// Pour cela on va créer 2 set, dans chaqune d'elles, on aura le
// changement du nom et la redéclaration du bind x pour appeler l'autre variable contenant l'autre nom
set nemesis1 "set name ^7Nemesis^3`^bR3; wait; bind x vstr nemesis2"
set nemesis2 "set name ^7n3mS.s^3`^bR3; wait; bind x vstr nemesis1"
bind x "vstr nemesis1"



toggle - avec toggle on permute entre 2 valeurs binaires (0 & 1). Ceci se révèle donc très utile si l'on souhaite faire des scripts simples, sans faire de set, etc.
Exemple:
// A l'aide de la touche x, je veux afficher ou pas mon arme
// On utilise donc cg_drawgun (0 = FALSE = Inactivé | 1 = TRUE = Activé)
bind x "toggle cg_drawgun"


echo - le texte qui suis est seulement affiché chez vous, au niveau des messages où vous voyez qui est mort ect..
Exemple:
bind x "set com_maxfps 125; echo <FPS 125>"

Seul les membres de votre équipe (axe/alliés) pourront le voir. Si vous mettez un texte apres un vsay, le texte apparaîtra accompagné du son.

Changer de FPS à l'aide d'un bouton
Objectif:
Je souhaite associer à ma touche 'n' le changement de FPS entre 46, 76 et 125, tout en affichant un message me prévenant que j'ai changé de FPS.

Démarche:
On va créer 3 variables contenant le changement de FPS et la redéclaration de la touche n.

Script:
set fps46 "set com_maxfps 46; echo ^3<FPS ^146^3>; bind n vstr fps76"

// Change le fps max à 76 et rebind n avec le contenu de fps125
set fps125 "set com_maxfps 125; echo ^3<FPS ^1125^3>; bind n vstr fps46"
// Change le fps max à 125 et rebind n avec le contenu de fps46
bind n "vstr fps46"

SpawnTimer:
Qu'est-ce qu'un spawntimer ?
Un spawntimer vous annonce les temps de respawn de vos adversaires.
A ma connaissance, il n'y a pas de spawntimer "connu" sur ET. J'éxpliquerais donc le principe du miens.

Un problème se pose quand à la conception de ce genre d'outils car (les spawntimer devant être internes) les boucles, les verifications de conditions, les calculs numériques, etc. n'éxistent pas pour nos cfg!
De plus, le temps de respawn des ennemis diffère selon les maps/config du serveur!

Principe:
A l'aide du clavier numérique, on tapera tout d'abord le temps de respawn des ennemis (ex: 30sec sur Oasis en config ClanBase).
Ensuite il faudra attendre de voir les ennemis "respawner" pour connaître un moment où ils "respawnent", et le tapper sur le clavier numérique.
On comprend tout de suite comment connaître les autres temps puisque le temps est cyclique.
Prenons par éxemple un temps de respawn toutes les 30sec. On les voit sortir du spawn à 00sec. On obtient alors un respawn tous les 00sec et 00+30=30sec (30+30=60, et 60sec=1min00sec, on retrouve bien nos 00sec du début =)).
On doit donc créer toutes les conditions possibles!

Démarche:
La touche 'kp_enter' servira à réinitialiser le spawntimer, il éxécutera le fichier 'timer.cfg' qui bind les touches 1 à 3 pour le premier digit du temps de respawn des ennemis (ex: l'ennemi a 20 sec de respawn, on tappe sur 2 pour avoir 2*).
La touche 1 correspond au fichier 1.cfg, 2 au fichier 2.cfg, 3 au fichier 3.cfg. Chacun de ces fichiers bind les touches 0 à 9 pour le second digit pour éxécuter les fichiers [premier digit][second digit].cfg (ex: si on a tapé 23, on éxécutera le fichier 23.cfg).
A ce stade, le spawntimer est configuré pour afficher les temps, il suffit alors de taper le temps où vous voyez vos ennmis respawn et le tour est joué.
C'est donc le même principe pour la secondes phase du spawntimer, on entre le premier et second digit, ce qui éxécutera un fichier qui affichera les temps et "bindera" la touche 'x' pour les réafficher.
Ce fichier est sous la forme [premier digit][second digit]_[premier digit'][second digit'] (ex: si on a tapé 15sec de respawn et on les a vu respawn à 00, le fichier 15_00.cfg sera éxécuté.

Script:
On va prendre l'exemple ci dessus.

timer.cfg

//Cette partie concerne l'initialisation du temps du respawn
//Quand on aura choisi son temps, le fichier correspondant sera executé
//Il faut l'ecrire avec deux digits, 02, 30, 25 ...
//Le temps que l'on pourra entrer sera compris entre 15 et 35 sec
echo ^7Timer^3`^bR3
echo ^7Entrez le temps de respawn de votre adversaire a l'aide du clavier numerique
echo ^7Pour annuler votre saisie, appuyez sur la touche Entr de votre clavier numerique

//**************************** BIND PREMIER DIGIT ******************************//
bind kp_end "exec r3timer/1.cfg; echo ^71*"
bind kp_downarrow "exec r3timer/2.cfg; echo ^72*"
bind kp_pgdn "exec r3timer/3.cfg; echo ^73*"

unbind x
unbind kp_ins
unbind kp_leftarrow
unbind kp_5
unbind kp_rightarrow
unbind kp_home
unbind kp_uparrow
unbind kp_pgup


1.cfg
//Premier digit: 1
bind kp_5 "exec r3timer/15.cfg; echo ^715 sec"
bind kp_rightarrow "exec r3timer/16.cfg; echo ^716 sec"
bind kp_home "exec r3timer/17.cfg; echo ^717 sec"
bind kp_uparrow "exec r3timer/18.cfg; echo ^718 sec"
bind kp_pgup "exec r3timer/19.cfg; echo ^719 sec"


15.cfg
//Second digit: 5
bind kp_end "exec r3timer/15_1.cfg; echo ^71*"
bind kp_downarrow "exec r3timer/15_2.cfg; echo ^72*"
bind kp_pgdn "exec r3timer/15_3.cfg; echo ^73*"
bind kp_leftarrow "exec r3timer/15_4.cfg; echo ^74*"
bind kp_5 "exec r3timer/15_5.cfg; echo ^75*"
bind kp_ins "exec r3timer/15_0.cfg; echo ^70*"


15_0.cfg
bind kp_ins "exec r3timer/15_00.cfg"
bind kp_end "exec r3timer/15_01.cfg"
bind kp_downarrow "exec r3timer/15_02.cfg"
bind kp_pgdn "exec r3timer/15_03.cfg"
bind kp_leftarrow "exec r3timer/15_04.cfg"
bind kp_5 "exec r3timer/15_05.cfg"
bind kp_rightarrow "exec r3timer/15_06.cfg"
bind kp_home "exec r3timer/15_07.cfg"
bind kp_uparrow "exec r3timer/15_08.cfg"
bind kp_pgup "exec r3timer/15_09.cfg"


15_00.cfg:
say_team ^9[^7Timer^3`^bR3^7: Enemy Respawn @ ^330 45 00 15^9]
echo ^70 sec, appuyez sur la touche ^3x ^7pour afficher les temps
set msg "say_team ^9[^7Timer^3`^bR3^7: Enemy Respawns @ ^330 45 00 15^9]"
bind x "vstr msg"

// Comme il arrive que l on se trompe ou bien que l'on ait pas un temps précis au début,
// on rebind les touches avec le contenu de 15.cfg,
// on peut donc rerentrer un temps sans tout refaire
bind kp_end "exec r3timer/15_1.cfg; echo ^71*"
bind kp_downarrow "exec r3timer/15_2.cfg; echo ^72*"
bind kp_pgdn "exec r3timer/15_3.cfg; echo ^73*"
bind kp_leftarrow "exec r3timer/15_4.cfg; echo ^74*"
bind kp_5 "exec r3timer/15_5.cfg; echo ^75*"
bind kp_ins "exec r3timer/15_0.cfg; echo ^70*"

unbind kp_rightarrow
unbind kp_home
unbind kp_uparrow
unbind kp_pgup
